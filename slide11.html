pre-ACDによる設計パラメータ管理

pre-Architecture Configuration Description - 制約条件の構造化手法

📋 pre-ACD とは

pre-Architecture Configuration Description
アーキテクチャ設計前に、ハードウェア制約・性能要求・リソース制限を
構造化YAML形式で記述し、生成AIに設計パラメータとして注入する手法
💾
センサーデータ処理システム の pre-ACD例
# センサーデータ処理システム pre-ACD project: name: "automotive_sensor_controller" version: "v2.1" target_platform: "ARM_Cortex_M7" hardware_constraints: memory: total: "64KB" stack: "8KB" heap: "16KB" static_data: "24KB" reserved: "16KB" timing: main_loop_cycle: "10ms" sensor_sampling_period: "1ms" actuator_response_max: "5ms" interrupt_latency_max: "50μs" peripherals: - adc: channels: 8 resolution: "12bit" sampling_rate: "1MHz" - timer: count: 4 precision: "1μs" max_frequency: "100MHz" - can_bus: bitrate: "500kbps" buffer_size: "128bytes" performance_requirements: real_time: deadline_miss_rate: "< 0.1%" wcet_margin: "20%" power: max_consumption: "2.5W" sleep_mode_current: "< 10μA" safety: iso26262_level: "ASIL-D" fault_detection_coverage: "≥ 95%"
🎯
生成AI活用の効果
  • 制約条件を明示的に定義: 曖昧さを排除
  • 設計パラメータとして注入: AIプロンプトに組み込み
  • 制約違反の事前回避: 不適切な設計選択を防止
  • 検証基準の明確化: 設計品質の客観的評価
📊

制約記述の標準化 30% → 95%

pre-ACDを活用した生成AI プロンプト例
/* 生成AIへの指示例 */ "以下のpre-ACD制約に従って、センサーデータ処理関数を生成してください。 制約条件: - メモリ使用量: 最大8KB以内 - 実行時間: 1ms以内（WCET保証） - 使用可能ADCチャンネル: 8ch、12bit解像度 - タイマー精度: 1μs - 安全要求: ASIL-D準拠（冗長化・故障検出必須） 生成要求: 1. ADCからのセンサー値読み取り関数 2. デジタルフィルタ処理（FIR、係数可変） 3. 異常値検出・アラート機能 4. CAN通信でのデータ送信 5. エラーハンドリング（故障診断対応） 出力形式: C言語、MISRA-C準拠"
結果: pre-ACDにより制約が明確になり、制約充足率が45%から89%に向上
